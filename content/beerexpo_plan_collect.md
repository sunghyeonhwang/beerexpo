# beerexpo.kr 콘텐츠 이전/재구성 작업 계획

## 1. 목표
- 기존 `https://beerexpo.kr/`의 콘텐츠를 새 사이트 제작에 바로 사용할 수 있도록 Markdown(`.md`) 기반으로 구조화한다.
- 단순 복사보다, 최신성/가독성/전환(CTA) 중심으로 재작성 가능한 기준 문서를 만든다.

## 2. 산출물
- `content-inventory.md`: 기존 사이트 페이지/섹션 자산 목록
- `sitemap.md`: 새 사이트 정보구조(IA)
- 페이지별 문서: `home.md`, `about-expo.md`, `exhibitors.md`, `program.md`, `visit.md`, `faq.md`, `contact.md` 등
- `migration-checklist.md`: 검수 체크리스트

## 3. 작업 범위 정의
- 전체 이전 여부 결정:
  - 옵션 A: 전체 페이지 이전
  - 옵션 B: 핵심 전환 페이지 우선 이전 (권장)
- 핵심 우선순위(권장):
  1. 홈
  2. 참가/방문 안내
  3. 프로그램/일정
  4. 참가사/부스
  5. FAQ/문의
  6. 공지/뉴스

## 4. 실행 단계
1. 원본 사이트 인벤토리 작성
- URL, 페이지 목적, 핵심 메시지, 콘텐츠 유형(텍스트/이미지/표/폼), 최신성, 유지/수정/삭제 여부를 표로 정리

2. 새 사이트 IA(사이트맵) 확정
- 사용자 과업 기준으로 메뉴 재배치
- 예시: `Home / Expo 소개 / 참가사 / 프로그램 / 방문안내 / 공지 / 문의`

3. Markdown 템플릿 표준화
- 모든 페이지를 동일 템플릿으로 작성해 운영 일관성 확보

4. 페이지별 초안 작성
- 우선순위 페이지부터 작성
- 원문 복붙 대신 새 톤앤매너로 재작성

5. 링크/자산 매핑
- 이미지, PDF, 외부 링크, 폼 연결 상태 점검
- 누락 자산 목록 별도 관리

6. QA 및 확정
- 최신성, 문장 품질, CTA 명확성, 모바일 가독성 검수 후 반영

## 5. Markdown 페이지 템플릿(권장)
```md
# 페이지명

## 목적
이 페이지가 달성해야 하는 비즈니스/사용자 목적

## 대상 사용자
주요 방문자 유형

## 핵심 메시지
사용자가 이 페이지에서 반드시 가져가야 할 메시지

## 본문 초안
실제 게시할 텍스트 초안

## CTA
- 1차 CTA:
- 2차 CTA:

## 필요 자산
- 이미지:
- 파일(PDF 등):
- 링크:

## SEO
- Title:
- Description:
- Primary Keywords:

## 상태/메모
- 상태: Draft / Review / Final
- 확인 필요 사항:
```

## 6. 검수 체크리스트
- 정보 최신성(날짜/장소/요금/참가 방법)
- 링크 유효성(404/리다이렉트)
- 용어 통일(행사명, 참가/참관 표현)
- CTA 명확성(버튼명, 이동 경로)
- 접근성(제목 구조, 대체 텍스트)
- 모바일 가독성(문장 길이, 리스트 활용)

## 7. 일정 제안(예시: 5일)
1. Day 1: 인벤토리 + 사이트맵 초안
2. Day 2: 핵심 3개 페이지 초안
3. Day 3: 나머지 페이지 초안 + 자산 매핑
4. Day 4: QA 1차 + 문구 정제
5. Day 5: 최종 검수 + 배포용 패키징

## 8. skills/에이전트 활용 계획
- 현재 확인된 로컬 스킬:
  - `skill-creator`: 새 스킬 설계/개선용
  - `skill-installer`: 외부/큐레이션 스킬 설치용
- 본 작업(콘텐츠 이전 계획 수립/MD 구조화)에는 위 스킬이 필수는 아님.
- 필요 시 활용 시나리오:
  1. 반복 작업 자동화가 필요하면 `skill-creator`로 "콘텐츠 인벤토리 생성 스킬" 정의
  2. 외부에서 검증된 웹 크롤링/콘텐츠 정리 스킬이 필요하면 `skill-installer`로 설치 검토
- 에이전트 역할 분리(권장):
  1. `Content Strategist`: IA/메시지/우선순위 정의
  2. `Content Editor`: 페이지별 카피 재작성
  3. `QA Reviewer`: 링크/최신성/표현 통일 검수
  4. `Publisher`: 최종 md 패키징 및 배포 연동

## 9. 리스크 및 대응
- 리스크: 원본 정보의 최신성 부족
  - 대응: 날짜 포함 팩트 우선 검증 후 반영
- 리스크: 페이지 간 중복 메시지
  - 대응: 페이지별 단일 목적 원칙 적용
- 리스크: 자료(이미지/PDF) 누락
  - 대응: 자산 매핑표와 누락 로그 동시 관리

## 10. 완료 기준(Definition of Done)
- 우선순위 페이지 md 작성 100%
- 사이트맵/인벤토리/체크리스트 문서 완료
- 주요 링크/자산 검수 완료
- 리뷰 반영 후 Final 상태 문서 확정
